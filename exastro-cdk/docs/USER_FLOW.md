## 前提条件
- 対象ユーザ: 製品開発部門の開発者


## Exastro CDKを使った開発フロー

1. カートリッジ開発
   - 顧客へ提供する製品とその運用を支える運用管理ツールを自動提供する仕組みをExastroコンポーネント（カートリッジ）として開発する
   - 主コンポーネント: Ansible Playbook + パラメータシート定義（SON）+ Conductor/Movement(JSON)
   - 追加コンポーネント: テスト用Conductor、Playbook、テストパラメータセット
2. カートリッジ検証・テスト
   - 検証環境で開発したコンポーネント一式をExastroへ一括登録
   - 製品＋運用管理ツール構築用Conductorを実行し、正常終了することを確認
   - ［追加要素］テスト用Conductorを実行し、構築対象間でエラーが発生していないか（監視状態が正しく設定されているか等）を確認
   - 失敗した場合はCDK上でコンポーネントを修正し実行
3. カートリッジ化
   - 検証・テストが完了したコンポーネントをExastroのkymファイル（カートリッジ）として出力
4. 本番適用（サービス提供）
   - 本番環境のExastroでkymをインポートし、顧客固有のパラメータを入力して実行
